# 📚 隐蔽小说阅读器 - 超详细打包教程（纯小白版）

> 本教程专为零基础用户设计，每一步都有详细说明，保证你能成功！

---

## 📋 准备工作

### 第 1 步：准备 GitHub 账号

**如果你还没有 GitHub 账号：**

1. 打开浏览器，访问：https://github.com
2. 点击右上角的绿色按钮 **"Sign up"**（注册）
3. 填写信息：
   - Email：填写你的邮箱（可以用 QQ 邮箱）
   - Password：设置密码（记住这个密码！）
   - Username：用户名（英文或数字，比如：xiaoming123）
4. 点击 **"Continue"** 继续
5. 填写一个个性化的名字，点击 **"Continue"**
6. 可能会让你验证邮箱，去邮箱里点击验证链接
7. 完成注册！

**如果你已经有 GitHub 账号：**
直接登录即可。

---

## 📤 第 2 步：创建代码仓库

1. 登录 GitHub 后，点击右上角的 **+** 号
2. 选择 **"New repository"**（新建仓库）
3. 填写仓库信息：
   - **Repository name**（仓库名称）：填写 `secret-reader` 或你喜欢的中文名
   - **Description**（描述）：填写 `隐蔽小说阅读器`
   - **Public / Private**：选择任意一个都可以（Public 公开，Private 私有）
4. 点击底部的绿色按钮 **"Create repository"**

✅ 仓库创建成功！

---

## 📥 第 3 步：下载 GitHub Desktop（推荐方法）

这是**最简单**的方法，不需要懂任何命令行！

### 3.1 下载 GitHub Desktop

1. 访问：https://desktop.github.com/
2. 点击 **"Download for Windows"** 下载
3. 下载完成后，双击安装文件
4. 按提示一步步安装，全部点 **"Next"** 或 **"Install"** 即可
5. 安装完成后，启动 GitHub Desktop

### 3.2 登录 GitHub Desktop

1. 启动 GitHub Desktop 后，点击 **"Sign into GitHub.com"**
2. 输入你的 GitHub 账号和密码
3. 点击 **"Sign in"** 登录
4. 可能需要验证，按提示操作即可

### 3.3 克隆仓库到本地

1. 在 GitHub Desktop 中，点击 **"File"** → **"Clone repository"**
2. 在列表中找到刚才创建的 `secret-reader` 仓库
3. 选择本地保存路径（建议选择桌面或文档）
4. 点击 **"Clone"**

⏳ 等待克隆完成（几秒钟）

### 3.4 复制代码到仓库

**现在你需要把小说阅读器的代码复制到仓库文件夹中：**

1. 打开文件夹：`C:\Users\你的用户名\Desktop\secret-reader`（或你选择的位置）
2. 打开另一个文件夹：`C:\Users\QYT-099\CodeBuddy\20260311165212`（刚才创建的项目文件夹）
3. 将 `20260311165212` 文件夹中的**所有文件**复制到 `secret-reader` 文件夹中
4. 确保复制以下文件：
   - ✅ `package.json`
   - ✅ `main.js`
   - ✅ `index.html`
   - ✅ `README.md`
   - ✅ `.gitignore`
   - ✅ `.github` 文件夹（整个文件夹）

**注意**：
- 不要复制 `node_modules` 文件夹（如果有的话）
- 确保所有文件都在 `secret-reader` 文件夹的根目录

### 3.5 提交并上传代码

1. 回到 GitHub Desktop
2. 你会看到左侧有文件变更列表
3. 在左下角的 **"Summary"** 框中输入：`初次提交`
4. 点击左下角的蓝色按钮 **"Commit to main"**
5. 点击后，点击右上角的 **"Push origin"** 按钮

✅ 代码上传成功！

---

## ⚙️ 第 4 步：等待自动打包

代码上传后，GitHub 会**自动开始打包**！

### 4.1 查看打包进度

1. 回到浏览器，打开你的 GitHub 仓库页面
2. 点击顶部的 **"Actions"** 标签（在仓库名称下方）
3. 你会看到 **"Build and Release"** 的任务正在运行
4. 点击这个任务查看详细进度

### 4.2 等待打包完成

⏱️ **打包需要 10-20 分钟**，请耐心等待！

你会看到：
- 黄色圆圈 ⏳ - 正在打包
- 绿色对勾 ✅ - 打包成功
- 红色叉 ❌ - 打包失败（如果失败，告诉我，我帮你解决）

### 4.3 下载打包好的文件

打包成功后：

1. 在 "Actions" 页面，点击完成的任务（绿色对勾）
2. 向下滚动页面，找到 **"Artifacts"** 部分
3. 你会看到一个文件：`secret-reader-windows`
4. 点击文件名右侧的下载图标（⬇️）
5. 浏览器会开始下载文件

⏳ 下载完成后，你得到一个 **.zip 压缩包**

---

## 💾 第 5 步：解压并安装

### 5.1 解压文件

1. 找到下载的压缩包（通常在"下载"文件夹）
2. 右键点击压缩包
3. 选择 **"解压到..."** 或 **"Extract All..."**
4. 选择解压位置（比如桌面）
5. 点击 **"确定"** 或 **"解压"**

解压后，你会看到一个 `.exe` 文件，比如：
- `隐蔽小说阅读器 Setup 1.0.0.exe`

### 5.2 安装软件

1. 双击 `.exe` 文件
2. 可能会弹出安全警告，点击 **"更多信息"** → **"仍要运行"**
3. 按安装向导一步步操作：
   - 选择安装位置（默认即可）
   - 点击 **"下一步"**
   - 点击 **"安装"**
   - 等待安装完成
   - 点击 **"完成"**

✅ 安装成功！桌面上会出现软件图标。

## 🚀 第 6 步：开始使用

### 启动软件

双击桌面上的 **"隐蔽小说阅读器"** 图标，软件就会启动！

### 基本操作

#### 1️⃣ 调节透明度

- 拖动窗口上方的 **透明度滑块**
- 向左拖动：窗口变透明（更隐蔽）
- 向右拖动：窗口变不透明（更清晰）
- 建议设置在 **70-85%** 之间

#### 2️⃣ 隐藏窗口（老板来了！）

有三种方法：

**方法 1：点击窗口控制栏**
- 点击窗口右上角的 **"◻"** 按钮

**方法 2：点击快速隐藏按钮**
- 点击窗口右下角的 **"🙈 快速隐藏"** 按钮

**方法 3：使用快捷键**
- 按键盘 `Alt` + `H`

✅ 窗口会隐藏，只在任务栏右下角有个小图标

#### 3️⃣ 显示窗口

当老板走了，你想继续看小说：

**方法 1：点击托盘图标**
- 在电脑右下角（系统托盘）找到图标
- 单击图标即可显示窗口

**找不到图标？**
- 点击右下角的 **^**（小箭头）展开隐藏图标
- 在里面找托盘图标

#### 4️⃣ 退出软件

**注意**：点击窗口的 × 按钮不会退出，只是隐藏！

**正确退出方法：**
1. 在托盘找到图标
2. **右键点击**图标
3. 选择 **"退出"**

#### 5️⃣ 切换章节

- 点击底部的 **"上一章"** / **"下一章"** 按钮
- 软件自带了示例小说，可以直接体验

#### 6️⃣ 置顶功能

- 点击 **"置顶"** 按钮，窗口会始终在最上层
- 再点一次可以取消置顶

## ✏️ 第 7 步：修改小说内容（可选）

如果你想换成自己的小说：

### 方法 1：直接编辑（简单）

1. 找到软件安装文件夹
   - 默认位置：`C:\Users\你的用户名\AppData\Local\Programs\secret-reader`
2. 找到 `index.html` 文件
3. 用记事本打开
4. 找到这段代码（约在第 200 行左右）：

```javascript
const chapters = [
    {
        title: "第一章 初遇",
        content: `这里是第一段内容。

这里是第二段内容。`
    },
    // ... 更多章节
];
```

5. 修改 `title`（章节标题）和 `content`（章节内容）
6. 保存文件
7. 重启软件即可

### 方法 2：替换整个文件（推荐）

1. 把你的小说按照章节格式整理好
2. 用文本编辑器打开 `index.html`
3. 替换整个 `chapters` 数组
4. 保存后重启

## ❓ 常见问题解答

### Q1: GitHub Desktop 登录失败怎么办？

**A:**
1. 确保网络连接正常
2. 尝试用浏览器登录 GitHub
3. 如果浏览器能登录，但 GitHub Desktop 不能，可能是网络问题，尝试：
   - 关闭防火墙或杀毒软件
   - 使用手机热点

### Q2: 代码上传后，Actions 没有自动运行？

**A:**
1. 检查 `.github/workflows/build.yml` 文件是否上传成功
2. 确保文件路径正确（必须是 `.github/workflows/` 这个完整路径）
3. 在仓库设置中，确保 Actions 已启用

### Q3: 打包失败（红色叉）怎么办？

**A:**
1. 点击失败的 Actions 任务
2. 查看错误信息
3. 最常见的原因：
   - `package.json` 格式错误（检查是否有多余的逗号）
   - 网络问题下载依赖失败
4. 如果自己解决不了，把错误信息截图发给我

### Q4: 找不到托盘图标？

**A:**
1. 点击 Windows 右下角的 **^**（小箭头）
2. 在弹出的图标列表中找
3. 如果还是找不到，可能是程序崩溃了，重新启动软件

### Q5: 软件启动失败？

**A:**
1. 检查是否被杀毒软件拦截
2. 关闭杀毒软件后重新安装
3. 以管理员身份运行

### Q6: 可以在 Mac 或 Linux 上使用吗？

**A:**
可以！但需要修改配置：

1. 打开 `.github/workflows/build.yml`
2. 找到这一行：
   ```yaml
   # os: [windows-latest, macos-latest, ubuntu-latest]
   ```
3. 改为：
   ```yaml
   os: [windows-latest, macos-latest, ubuntu-latest]
   ```
4. 提交后重新打包，会自动生成三个平台的版本

### Q7: 打包后的文件在哪里？

**A:**
在 GitHub 的 Actions 页面下载，步骤见第 4.3 节。

### Q8: 可以同时打包多个版本吗？

**A:**
可以！修改 `.github/workflows/build.yml`，取消注释 Mac 和 Linux 的配置即可。
